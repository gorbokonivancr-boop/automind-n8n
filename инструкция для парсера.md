# Инструкция по управлению парсером контактов HeadHunter

Данное руководство поможет вам самостоятельно настраивать и контролировать работу системы сбора данных. Парсер автоматически находит вакансии на HH.ru, извлекает контакты (телефоны, почту, ИНН) как из описания вакансии, так и с официальных сайтов компаний, и сохраняет их в Google Таблицу.

---

## 1. Где живут данные и как они обновляются

Все собранные контакты попадают в вашу [Google Таблицу](https://docs.google.com/spreadsheets/d/1zknVZMtz2osTrEoVbQDN1jWyVQSPybVE54FNlTlVhIY).

*   **Вкладка «Контакты»**: основной накопительный отчет. Сюда попадают данные по широкому списку профессий за последние 10-30 дней.
*   **Вкладки с датами (например, `2026-05-14`)**: ежедневный «срез». Сюда попадают только самые свежие вакансии, опубликованные за последние 24 часа.

## 2. Как подключиться к серверу (управлению)

Парсер работает на удаленном компьютере (сервере) по адресу: **31.129.98.155**.

1.  Откройте программу «Терминал» (Mac/Linux) или «PowerShell» (Windows).
2.  Введите команду: `ssh root@31.129.98.155`
3.  Введите пароль (он не будет отображаться при вводе, это нормально). Нажмите **Enter**.

Теперь вы находитесь в системе и можете вносить изменения.

## 3. Настройка фильтров и параметров (Файл .env)

Все настройки находятся в одном файле — `.env`. Чтобы его отредактировать, перейдите в папку проекта и откройте редактор:

```bash
cd /opt/hh-parser
nano .env
```

### 3.1. Основные фильтры поиска:

| Параметр | Что это значит | Пример значения |
| :--- | :--- | :--- |
| **HH_JOB_QUERIES** | **Список профессий**. Ключевые слова для поиска. Можно использовать сложные связки (см. ниже). | `"Маркетолог|Бизнес-ассистент|Менеджер"` |
| **HH_AREA_IDS** | **География (Регионы)**. Коды городов/стран через запятую. `113` — вся Россия, `1` — Москва, `2` — Санкт-Петербург. | `1,2` (поиск только по МСК и СПб) |
| **HH_SEARCH_PERIOD_DAYS** | **Глубина поиска**. За какой период (в днях) проверять вакансии. | `10` (оптимально), `30` (максимум) |
| **HH_TARGET_CONTACTS** | **Лимит строк**. Сколько контактов нужно найти за один цикл. | `5000` |
| **REMOTE_ONLY** | **Удаленная работа**. `1` — искать только удаленку, `0` — искать все вакансии (офис, гибрид и т.д.). | `1` |
| **HH_WRITE_EMPTY** | **Фильтр "Пустых"**. `0` — записывать только если найден телефон/email, `1` — писать всех подряд. | `0` (рекомендуется) |
| **HH_MAX_SEARCH_PAGES** | **Глубина страниц**. Сколько страниц поиска HH просматривать (макс. 40). | `30` |

### 3.2. Технические ключи (если перестало работать):
Если сбор данных прекратился, возможно, нужно обновить API-ключи HeadHunter (создаются на [dev.hh.ru](https://dev.hh.ru)):
- `HH_CLIENT_ID` / `HH_CLIENT_SECRET` — основные ключи.
- `HH_CLIENT_ID_DAILY` / `HH_CLIENT_SECRET_DAILY` — ключи для ежедневного отчета.

**Совет по редактору:** После внесения изменений нажмите `Ctrl + O`, затем `Enter` (для сохранения) и `Ctrl + X` (для выхода из редактора).

### 3.3. Лайфхаки для поиска (в параметре HH_JOB_QUERIES):
Вы можете использовать логику самого HeadHunter прямо в кавычках:
*   **Исключение слов**: `"Менеджер NOT Продажи"` — найдет менеджеров, но пропустит тех, у кого в названии есть «Продажи».
*   **Точное совпадение**: `"!Дизайнер"` — восклицательный знак перед словом заставит искать именно эту форму слова (не найдет «Дизайн»).
*   **Поиск фразы**: `"\"Помощник руководителя\""` — кавычки внутри (с косой чертой `\"`) найдут точное словосочетание.

## 4. Как применить новые настройки

Чтобы парсер начал работать по новым правилам, его нужно перезапустить.

1.  **Остановите текущий процесс:**
    ```bash
    pkill -f parser_final_fixed.py
    ```
2.  **Запустите основной сборщик:**
    ```bash
    cd /opt/hh-parser
    nohup ./venv/bin/python parser_final_fixed.py > main_contacts.log 2>&1 &
    ```

Парсер снова работает в фоновом режиме, даже если вы закроете окно терминала.

## 5. Работа с данными: Фильтры и сценарии поиска

Ваша Google Таблица — это не просто список, а активная база для лидогенерации. Чтобы извлекать из нее максимум пользы, используйте встроенные инструменты фильтрации.

### 5.1. Как включить фильтры (если они выключены)
1. Выделите верхнюю строку (шапку) с названиями колонок.
2. В верхнем меню Google Таблиц нажмите: **Данные** -> **Создать фильтр**.
3. В каждой ячейке шапки появится значок «воронки».

### 5.2. Примеры полезных фильтров для бизнеса:

**Сценарий А: «Только горячие контакты»**
*   **Задача**: Убрать компании, у которых нет телефона или почты.
*   **Действие**: Нажмите на воронку в колонке **«Телефон»**, выберите «Фильтровать по условию» -> «Текст не пуст». Повторите для колонки **«Email»**.
*   **Результат**: Перед вами список компаний, которым можно звонить или писать прямо сейчас.

**Сценарий Б: «Работа с крупным бизнесом»**
*   **Задача**: Отсеять мелкие ИП и оставить только серьезные организации.
*   **Действие**: В колонке **«Тип»** снимите галочку с «ИП».
*   **Результат**: В списке останутся только ООО, АО и ПАО.

**Сценарий В: «Географический охват»**
*   **Задача**: Найти вакансии только в конкретном городе (например, Казань).
*   **Действие**: В колонке **«Город»** нажмите «Очистить» и введите в поиске «Казань», отметьте её галочкой.

**Сценарий Г: «Анализ свежести»**
*   **Задача**: Увидеть только те компании, которые мы нашли за последние 2 дня.
*   **Действие**: В колонке **«Дата»** выберите сортировку «Я-А» (от новых к старым) или используйте «Фильтр по условию» -> «Дата после...».

**Сценарий Д: «Поиск по ключевым словам в описании»**
*   **Задача**: Найти компании, которые упоминают конкретные технологии или условия (например, "English" или "Relocation").
*   **Действие**: В колонке **«Вакансия»** выберите «Фильтровать по условию» -> «Текст содержит» -> введите нужное слово.

### 5.3. Продвинутые инструменты (Срезы и Фильтры для команды)

Если с таблицей работают несколько человек одновременно, обычные фильтры могут мешать друг другу.
*   **Фильтр для себя**: Нажмите **Данные** -> **Режимы фильтра** -> **Создать новый режим фильтра**. Теперь то, что вы отфильтруете, будете видеть только вы.
*   **Срезы (Кнопки-фильтры)**:
    1. Нажмите **Вставка** -> **Срез**.
    2. Выберите колонку (например, «Город»).
    3. В таблице появится плавающее окошко-кнопка. Это очень удобно для быстрой фильтрации одним кликом без захода в меню колонок.

### 5.4. Автоматическая раскраска (Условное форматирование)
Вы можете заставить таблицу саму подсвечивать важные строки:
1. Выделите всю таблицу (нажмите на пустой квадрат в углу между `1` и `A`).
2. Меню: **Формат** -> **Условное форматирование**.
3. **Правило**: «Текст содержит» слово `HH+site`.
4. **Стиль**: Выберите зеленый фон.
*   **Зачем**: Теперь вы сразу видите компании, у которых парсер нашел контакты не только в вакансии, но и просканировал их официальный сайт. Это самые качественные данные.

## 6. Расшифровка колонок:

В таблицу выгружается 16 колонок. Вот что они означают:

1.  **Дата**: когда была добавлена строка.
2.  **Компания**: название организации.
3.  **Вакансия**: название должности.
4.  **Город**: где находится офис (или город вакансии).
5.  **Телефон**: номера, найденные в вакансии или на сайте.
6.  **Email**: адреса электронной почты.
7.  **Сайт**: ссылка на официальный сайт компании.
8.  **Соцсети**: ссылки на VK, Telegram, WhatsApp и др.
9.  **ИНН**: налоговый номер компании (полезно для проверки надежности).
10. **Тип**: юридическая форма (ООО, ИП, АО).
11. **Зарплата**: указанная вилка (от и до).
12. **Опыт**: требования к стажу.
13. **График**: удаленка, офис, гибрид.
14. **Тип занятости**: полная занятость, стажировка и т.д.
15. **Ссылка HH**: прямая ссылка на вакансию.
16. **Источник**: откуда взяты контакты («HH» или «HH+site»).

## 7. Что делать, если возникли проблемы?

### Если таблица не обновляется:
1.  **Проверьте, работает ли программа:**
    Введите команду `ps aux | grep parser_final`. Если видите строки со словом `python`, значит всё в порядке.
2.  **Загляните в «черный ящик» (Логи):**
    Программа записывает всё, что делает. Посмотрите последние 30 действий:
    ```bash
    tail -30 /opt/hh-parser/main_contacts.log
    ```
    Если там написано `skip duplicate`, значит парсер находит компании, которые уже есть в вашей таблице.

### Если видите ошибки доступа:
Проверьте, не удалили ли вы случайно доступ для технического почтового ящика (сервисного аккаунта) к вашей Google Таблице. Его адрес указан в файле `service_account.json`.

## 8. Полная автоматизация (Cron)

Система настроена так, что вам не нужно ничего запускать вручную:
*   **Каждое утро в 07:50** — парсер перезагружается, чтобы обновить данные.
*   **Каждое утро в 08:00** — запускается ежедневный срез (новая вкладка с датой).
*   **Раз в неделю** — система сама чистит старые логи, чтобы на сервере не закончилось место.
*   **При перезагрузке сервера** — парсер включается автоматически через 30 секунд.
